# 比特币白皮书 — 第 2-3 节精要

来源：https://bitcoin.org/bitcoin.pdf (中英文 PDF 已下载到 `references/` 目录)

## 第 2 节：交易 (Transactions)

### 核心概念
- 一枚电子货币被定义为一串**数字签名链（chain of digital signatures）**
- 每个所有者通过对前一笔交易的哈希和下一个所有者的公钥进行**数字签名**，将货币转移给下一个所有者
- 收款人可以验证签名链来验证所有权

### 工作流程
```
交易 A: Owner1 用 Owner2 的公钥锁定一个 UTXO
       ↓ Owner1 的签名
交易 B: Owner2 用 Owner3 的公钥锁定该 UTXO
       ↓ Owner2 的签名
交易 C: Owner3 用 Owner4 的公钥锁定该 UTXO
```

### 关键问题：双重支付 (Double-Spending)
- 收款人无法验证某个所有者是否将同一笔货币双重支付给了其他人
- 传统方案：引入可信第三方（银行）来仲裁
- 比特币的方案：用第 3 节的时间戳服务器 + 第 4 节的工作量证明来解决

## 第 3 节：时间戳服务器 (Timestamp Server)

### 核心概念
时间戳服务器通过对**一个区块（block）中的项目进行哈希**并发布该哈希，为这些项目打上时间戳。每个时间戳在其哈希中包含前一个时间戳，形成一条**链（chain）**。

### 数据结构
```
区块 A: [Tx₁, Tx₂, Tx₃] → Hash(A)
                             ↓ 包含
区块 B: [Tx₄, Tx₅, Tx₆] → Hash(B) (包含 Hash(A))
                             ↓ 包含
区块 C: [Tx₇, Tx₈, Tx₉] → Hash(C) (包含 Hash(B))
```

### 为什么这能解决双重支付？
- 一旦某个交易被打包进区块并被哈希，要修改它就必须重新计算**该区块之后所有区块**的哈希
- 由于每个新区块都包含前一个区块的哈希，形成了**防篡改链**
- 这就是"区块链"这个名字的由来
