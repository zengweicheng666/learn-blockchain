# 哈希函数（SHA256）参考资料

## 哈希的四个性质

| 性质 | 含义 | 在区块链中的应用 |
|------|------|-----------------|
| **确定性** | 同一输入永远产生同一输出 | 任意两个节点算同一个区块的哈希 → 结果一致 |
| **快速计算** | 输入任意长度数据都能快速得到哈希 | 矿工/验证者可以快速验证区块 |
| **雪崩效应** | 输入微变 → 输出巨变（~50% bits 不同） | 篡改交易 → 哈希全变 → 链条断裂 |
| **抗碰撞性** | 无法找到两个不同输入产生相同哈希 | 保证"哈希即身份"的安全性 |

## 为什么叫 SHA256？

- **SHA** = Secure Hash Algorithm（安全哈希算法）
- **256** = 输出固定为 **256 bits**（即 64 个十六进制字符）
- 属于 SHA-2 家族，目前尚未被有效攻破

## 区块链里哪些地方用了哈希？

```
1. 区块链接       ←  parent_hash 字段
2. 交易指纹       ←  每笔交易的 tx hash
3. Merkle Tree    ←  逐层哈希汇总
4. 工作量证明 PoW ←  找 Nonce 使哈希满足条件
5. 地址生成       ←  公钥 → Keccak256 → 地址
6. 数字签名       ←  先对消息取哈希再签名
```
