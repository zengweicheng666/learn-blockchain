# 分叉与最长链原则参考资料

## 一、临时分叉（自然分叉）

### 产生原因

两个矿工几乎同时挖出有效区块，都广播到网络：

```
         ┌── Block A ── Block C ── Block E（累计 PoW 最多 → 主链）
         │
创世区块 ──┤
         │
         └── Block B ── Block D（累计 PoW 较少 → 孤块/叔块）
```

### 解决方式：最长链原则

- 网络中的节点总是选择**累计工作量最多（PoW）**或**累计证明最多（PoS）**的那条链
- 一旦 C 和 D 同时出现，矿工会在先收到的那个块上继续挖
- 当其中一个分支率先出了下一个块（E），它就更长 → 全网切换过去
- 短链上的区块变成**叔块（Uncle Block）/ 孤块**

### 叔块（Uncle Block）

- 以太坊特有的机制（比特币没有，比特币的孤块直接丢弃）
- 以太坊 PoW 时代，叔块仍然获得部分奖励（~87.5% 的全额奖励）
- 目的：鼓励去中心化，减轻网络延迟对小型矿工的惩罚

### "6 个确认"的含义

- 你的交易被打包进区块后，后面又出了 6 个新区块
- 要回滚你的交易，攻击者需要重算这 7 个区块的 PoW
- 6 个确认 ≈ 交易基本不可逆

**各链不同的确认数：**
| 链 | 出块时间 | 建议确认数 | 等待时间 |
|----|---------|-----------|---------|
| 比特币 | 10 min | 6 | ~60 min |
| 以太坊 | ~12-15s | 12~24 | ~3-5 min |
| 交易所（大额） | — | 30+ | 视链而定 |

## 二、硬分叉（Hard Fork）

### 定义

协议规则发生**不向后兼容**的更改。新旧节点无法认同对方的区块。

### 典型例子

**以太坊 DAO 硬分叉（2016）：**
- 攻击者利用 DAO 合约漏洞盗走 360 万 ETH
- 社区分裂：升级代码回滚被盗资金（新链） vs 认为代码即法律（旧链）
- 结果：以太坊（ETH） vs 以太经典（ETC）—— 两条链至今共存

```
                   ┌── ETH（升级，回滚被盗资金）
以太坊原始链 ──────┤
                   └── ETC（拒绝升级，"Code is Law"）
```

### 硬分叉的特点
- 需要全节点升级软件
- 持币者自动获得分叉后的两种币
- 会造成社区分裂

## 三、软分叉（Soft Fork）

### 定义

协议规则发生**向后兼容**的更改。旧节点仍认可新节点产生的区块。

### 例子

- 比特币 SegWit（隔离见证）：交易格式变了，但旧节点仍能验证
- 旧节点看到新区块：虽然看不懂某些新字段，但认为区块格式合法

### 与硬分叉的区别
| | 硬分叉 | 软分叉 |
|--|--------|--------|
| 兼容性 | 不向后兼容 | 向后兼容 |
| 节点升级 | 全部必须升级 | 旧节点仍可用 |
| 分裂风险 | 高（可能永久分裂） | 低 |
| 激活难度 | 需全网共识 | 只需多数矿工支持 |

## 四、51% 攻击

### 攻击如何实施

攻击者控制 >50% 的算力（PoW）或质押（PoS）：

```
正常链:   ... → A → B → C → D → E → F（公开的）
攻击链:   ... → A → B → C' → D' → E' → F' → G'（私下挖，不回滚自己的交易）
```

攻击者秘密挖一条更长的链，然后广播 → 全网切换 → 攻击者在原链上的交易被回滚（双重支付）

### 攻击能做 ✓
- 回滚自己的交易（双重支付）
- 阻止特定交易被打包（审查）
- 重排交易顺序

### 攻击不能做 ✗
- 修改他人的交易（没有私钥）
- 凭空创造新币
- 窃取他人资金

### 实际案例

- **Bitcoin Gold（2018）：** 攻击者控制算力，双重支付，损失约 1800 万美元
- **ETC（2020）：** 多次 51% 攻击，总计损失数百万美元

### 防御措施

- 确认数越多越安全（大额交易等 30+ 确认）
- 监控算力分布，发现异常自动暂停
- 转向 PoS（以太坊的选择——攻击成本 = 质押的 ETH，可以被罚没/slash）
