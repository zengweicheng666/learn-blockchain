# PoW 工作量证明参考资料

## 一句话

PoW 就是"你证明你花了力气"。在区块链里：找到一个 Nonce 使得 `SHA256(区块头)` 满足难度条件。

## 挖矿流程图

```
                     ┌──────────────────────┐
                     │  打包交易组成区块      │
                     └────────┬─────────────┘
                              ↓
                     ┌──────────────────────┐
                     │  设 Nonce = 0         │
                     └────────┬─────────────┘
                              ↓
                     ┌──────────────────────┐
                     │  计算 SHA256(区块头)   │
                     └────────┬─────────────┘
                              ↓
                     ┌──────────────────────┐
              ┌─────│  哈希满足难度？         │
              │     └────────┬─────────────┘  │
              │              ↓                 │
              │           ┌───┴───┐            │
              │        是 │       │ 否          │
              │           ↓       ↓            │
              │     ┌────────┐ ┌────────┐      │
              │     │ 广播区块 │ │Nonce+1 │     │
              │     │  获得奖励│ │ 继续算  │     │
              │     └────────┘ └────────┘      │
              │                                │
              └────────────────────────────────┘
```

## 难度调整

比特币每 2016 个区块（约 2 周）调整一次难度：

```
当前出块时间 20160 min (10 min × 2016)
     ↓
如果实际出块时间 < 20160 min → 全网算力变强 → 调高难度
如果实际出块时间 > 20160 min → 全网算力变弱 → 调低难度
     ↓
目标：让出块时间稳定在 10 分钟
```

## 现实中的难度

| 链 | 出块时间 | 共识 | 当前难度（约） |
