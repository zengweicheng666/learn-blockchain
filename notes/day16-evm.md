## 一、EVM 是什么？

> EVM = 全球统一的**分布式状态机**。所有以太坊节点运行同样的 EVM，执行同样的代码，得到同样的结果。

```
                  Solidity 代码
                       │
                    solc 编译
                       │
                  ┌─────▼─────┐
                  │   字节码    │  ← hex 字符串
                  └─────┬─────┘
                       │
                  ┌─────▼─────┐
                  │   Opcodes  │  ← 约 140 条指令
                  └─────┬─────┘
                       │
                  ┌─────▼─────┐
                  │   EVM 执行  │
                  └───────────┘
```

### 关键特性

| 特性 | 说明 |
|------|------|
| **确定性** | 相同输入 → 所有节点相同输出 |
| **沙箱隔离** | 每个合约有独立存储空间，互不干扰 |
| **准图灵完备** | 可执行任意计算，但有 Gas 限制 |
| **栈式架构** | 所有计算通过栈完成（非寄存器） |

---

## 二、EVM 三大数据区域

| 区域 | 生命周期 | 特点 | 类比 |
|------|----------|------|------|
| **Stack（栈）** | 执行期间 | LIFO，最大深度 1024，每项 256 位 | 草稿纸 |
| **Memory（内存）** | 单次交易 | 线性可寻址字节数组，用完清除 | 白板 |
| **Storage（存储）** | **永久** | 256 位键值对，写入成本极高 | 硬盘 |

### 执行示例

```
PUSH1 0x05    // 栈: [5]
PUSH1 0x03    // 栈: [5, 3]
ADD           // 弹出 3+5，压入 8 → 栈: [8]
```

## 三、Gas

### 为什么需要 Gas？

> 图灵完备 = 理论上可能有无限循环。Gas 确保：**你可以跑任何程序，但必须为每一步付费**。

### 常见操作 Gas 成本

| 操作 | Gas | 类比 |
|------|-----|------|
| 基础转账 | 21,000 | 寄一个快递 |
| ADD / SUB | 3 | 按一下计算器 |
| SLOAD（读存储） | ~800 | 打开一个文件 |
| SSTORE（首次写） | ~20,000 | 刻一个石雕 |
| SSTORE（改值） | ~5,000 | 修改石雕 |

### 费用公式

```
交易费 = Gas Used × Gas Price
```

Gas 耗尽 → 状态全部回滚，但 Gas **不退**。

## 四、Opcode 分类

| 类别 | 示例 | Gas 成本 |
|------|------|---------|
| 算术/逻辑 | ADD, MUL, SUB, AND, OR | 3–10 |
| 栈操作 | PUSH, POP, DUP, SWAP | 3 |
| 内存 | MLOAD, MSTORE | 低–中 |
| 存储 | SLOAD, SSTORE | 高（800–20,000） |
| 控制流 | JUMP, JUMPI, STOP, RETURN, REVERT | 低 |
| 环境 | CALLER, CALLVALUE, TIMESTAMP, BLOCKHASH | 中 |
| 外部调用 | CALL, STATICCALL, DELEGATECALL | 中–高 |

## 五、EVM 沙箱模型

```
┌─────────────────────────────┐
│        全局状态              │
│  ┌──────┐  ┌──────┐        │
│  │合约 A │  │合约 B │  ...  │
│  │storage│  │storage│       │
│  └──────┘  └──────┘        │
│         ▲                   │
│         │ 外部调用           │
│         ▼                   │
│  ┌──────────────────┐       │
│  │  EVM 执行环境      │      │
│  │  Stack/Memory/    │      │
│  │  Storage(当前合约) │      │
│  └──────────────────┘       │
└─────────────────────────────┘
```

每个合约只能读写自己的 Storage。外部调用（CALL）会创建新的执行上下文，不能直接修改调用者的存储。

## 疑问记录

1. DELEGATECALL 和 CALL 有什么区别？——CALL 在被调用者的上下文中执行（用自己的 storage），DELEGATECALL 在调用者的上下文中执行（用调用者的 storage）。DELEGATECALL 是实现"代理合约/可升级合约"的关键
2. 同一个合约的多次调用之间，Storage 会保留吗？——会！Storage 是永久存储，写入后除非被修改或合约自毁，否则永久保存在链上
