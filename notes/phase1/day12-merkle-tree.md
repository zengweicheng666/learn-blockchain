## 一、什么是默克尔树？

> 一种基于哈希的二叉树结构：叶子是数据哈希，父节点是子节点哈希拼接后再哈希，最终汇聚成一个根哈希。

```
         Root = hash(H1 + H2)
        /                    \
     H1 = hash(A+B)        H2 = hash(C+D)
     /        \            /        \
    A          B          C          D
  (tx1)      (tx2)      (tx3)      (tx4)
```

### 核心特性

| 特性 | 说明 |
|------|------|
| **防篡改** | 任意叶子改变 → 根哈希完全变化 |
| **高效验证** | 验证一个叶子只需 O(log n) 个兄弟哈希 |
| **伸缩性** | 无论 10 条还是 100 万条交易，Proof 都是 20-40 个哈希 |

## 二、Merkle Proof

### 问题

轻节点（如手机钱包）不可能下载整个区块的所有交易来验证一笔交易是否存在。

### 解决方案

全节点提供一个 **Merkle Proof**——只需包含从该叶子到根路径上的所有**兄弟哈希**，轻节点用 O(log n) 步计算就能验证。

```
要证明 tx3 在树中：
Path: C → H2 → Root
Proof = [D (右兄弟), H1 (左兄弟)]

验证过程：
hash(C) → 与 D 拼接哈希 → H2 → 与 H1 拼接哈希 → Root
最终 hash == root ✅
```

### 空间节省

| 交易数 | 树深度 | Proof 大小 |
|--------|--------|-----------|
| 100 | 7 | 7 个哈希 ≈ 224 字节 |
| 10,000 | 14 | 14 个哈希 ≈ 448 字节 |
| 1,000,000 | 20 | 20 个哈希 ≈ 640 字节 |

## 三、安全考虑：第二原像攻击

如果不加区分地用同样的方式哈希叶子和内部节点，攻击者可以伪造证明：

```
叶子：hash("tx1")
内部：hash("tx1" + "tx2")   // 和叶子用同样的 hash 函数
```

如果攻击者构造一个"叶子"，内容刚好等于某个内部节点的子节点哈希拼接，这个假叶子就能通过验证。

**防御措施**：

| 做法 | 说明 |
|------|------|
| 叶子加 0x00 前缀 | `hash(0x00 + data)` |
| 内部加 0x01 前缀 | `hash(0x01 + left + right)` |
| 叶子和内部用不同哈希函数 | 增加实现复杂度，不常用 |
