## 一、DPoS（委托权益证明）

### 核心思想

> 代币持有者投票选出有限数量的"代表/见证人（超级节点）"，由他们负责出块和验证。

```
持币人投票 → 选出超级节点 → 超级节点轮流出块
                         → 不称职者被投票替换
```

### 代表项目与参数

| 项目 | 超级节点数量 | 特点 |
|------|-------------|------|
| EOS | 21 个 + 100 备用 | 每轮 21 个区块，前 20 按得票，第 21 按概率 |
| TRON | 27 个 | 类似 EOS 的 DPoS 变体 |
| BitShares | 20-100 可变 | DPoS 概念的起源项目（Daniel Larimer 发明） |

### 优点

- **吞吐量高**——TPS 可达数千到百万级，远超 PoW/PoS
- **交易费极低**——无需算力竞赛，出块成本低
- **确认快**——EOS 约 3 秒出块

### 缺点

- **中心化倾向**——21 个超级节点容易被合谋或寡头化
- **投票参与度低**——多数持币人不参与投票，实际控制权集中
- **背离"无许可"精神**——普通节点无法参与出块

## 二、PBFT（实用拜占庭容错）

### 核心思想

> 通过多轮投票（Pre-prepare → Prepare → Commit），在存在恶意节点的网络中达成共识。

```
Leader 提议 → 所有节点 Pre-prepare → Prepare 广播 → Commit 确认 → 执行
```

### 容错模型

> 在 `3f + 1` 个节点的系统中，最多容忍 `f` 个节点作恶。
> 例如：7 个节点时最多容忍 2 个恶意节点。恶意节点 ≥ 1/3 时系统不安全。

### 代表项目

> Hyperledger Fabric（早期版本）、Stellar、FISCO BCOS、Zilliqa

### 三阶段流程

1. **Pre-prepare（预准备）**：Leader（主节点）向所有 Replica（副本节点）广播提案，包含区块内容和编号
2. **Prepare（准备）**：每个副本验证提案后向全网广播 <Prepare> 消息。当节点收到 `2f + 1` 个 Prepare（含自己）后进入 Commit 阶段
3. **Commit（提交）**：节点广播 <Commit> 消息。当收到 `2f + 1` 个 Commit 后，执行区块操作，达成最终共识

### 优缺点

| 优点 | 缺点 |
|------|------|
| 秒级最终确认，无需等待概率确认 | 通信复杂度 O(n²)，节点越多越慢 |
| 数学上可证明的拜占庭容错 | 节点数通常 ≤ 100，扩展性差 |
| 低延迟，适合联盟链场景 | Leader 更换流程（View Change）复杂 |

## 三、CAP 定理

> 分布式系统最多只能同时满足三个特性中的两个。

### 三个特性

| 特性 | 含义 | 类比 |
|------|------|------|
| 一致性（Consistency） | 所有节点在同一时刻看到相同的数据 | 银行账户——转账后所有 ATM 余额一致 |
| 可用性（Availability） | 每个请求都能收到应答（成功或失败） | 网站 7×24 小时不宕机 |
| 分区容忍性（Partition Tolerance） | 网络断开时系统仍能工作 | 海底光缆断了，两边还能分别运行 |

### 区块链的 CAP 取舍

| 类型 | 选择 | 原因 |
|------|------|------|
| 公链（Bitcoin/Ethereum） | AP | 必须接受 P（分布式），保证 A（持续出块），放弃强一致性 → 概率最终性 |
| 联盟链（Hyperledger Fabric） | CP | 必须接受 P，保证强一致性，牺牲部分可用性（节点少，网络稳定时才工作） |

## 四、共识机制全景对比表

| 维度 | PoW | PoS | DPoS | PBFT |
|------|-----|-----|------|------|
| 代表项目 | Bitcoin, 以太坊旧版 | 以太坊现行, Cosmos | EOS, TRON, BitShares | Hyperledger Fabric, Stellar |
| 网络类型 | 公链（无许可） | 公链（无许可） | 公链（无许可） | 联盟链（许可制） |
| 最终性 | 概率最终性（6 块 ≈ 不可逆） | 经济最终性（≥2/3 质押投票） | 经济最终性 + BFT 混合 | 绝对最终性（数学证明） |
| 吞吐量（TPS） | BTC: 7, ETH: 15 | ETH: ~30 | 1000 ~ 百万级（理论） | 数千 |
| 节点规模 | 无限（任何人可加入） | 无限（任何人可质押） | 有限（21-100 个超级节点） | ≤ 100 个共识节点 |
| 能耗 | 极高（挖矿） | 低（质押） | 极低 | 极低 |
| 去中心化程度 | 高（但矿池集中化） | 高（但质押池集中化） | 低（超级节点模式） | 低（许可制） |
| 容错能力 | 算力 ≤ 50% | 质押 ≤ 50%（经济惩罚加重） | 投票替换恶意节点 | 数学保证 ≤ 33% 恶意节点 |
| 适用场景 | 最大程度去中心化 | 公链基础设施 | 高性能 DApp | 企业/机构联盟链 |
