## 一、Gas 机制基础

### 三个核心概念

| 概念 | 含义 | 类比 |
|------|------|------|
| **Gas** | 计算量的**计量单位** | 汽油的"升" |
| **Gas Limit** | 你愿意为这笔交易支付的 **Gas 上限** | 油缸容量 |
| **Gas Price** | 每单位 Gas 你愿意付多少钱 | 每升油价 |

### 费用公式（EIP-1559 前）

```
交易费 = Gas Used × Gas Price
```

### 费用公式（EIP-1559 后）

```
交易费 = Gas Used × (Base Fee + Priority Fee)
         maxFeePerGas ≥ Base Fee + Priority Fee
```

## 二、EIP-1559 改革（2021 年 8 月）

### 为什么改革？

旧模式的问题：用户设置一个 `gasPrice`，矿工优先打包出价高的交易——导致 Gas 费像拍卖一样飙升，用户很难预估该出多少。

### 改革后的两段式费用

```
用户支付 = Base Fee（燃烧）  +  Priority Fee（给验证者）
              │                          │
        协议自动调整                 用户自愿给小费
        全部销毁                     加速交易
```

### Base Fee 自动调节机制

```
区块 Gas 用量 > 15M  →  Base Fee 上调（最多 +12.5%）
区块 Gas 用量 < 15M  →  Base Fee 下调
区块 Gas 用量 = 15M  →  Base Fee 不变
```

### 效果

- 网络拥堵 → Base Fee 上涨 → 部分用户暂缓交易 → 拥堵自然缓解
- 用户不再需要猜价——只需设置 `maxFeePerGas` 作为保底上限

## 三、Base Fee 销毁的经济影响

```
          PoW 时期                     PoS + EIP-1559 时期
  ┌─────────────────────┐     ┌──────────────────────────┐
  │ 每日发行 ~13,000 ETH │     │ 每日发行 ~1,700 ETH       │
  │ + 交易费归矿工       │     │ + Base Fee 全部销毁       │
  │ = 持续通胀           │     │ = 活跃期**净通缩**       │
  └─────────────────────┘     └──────────────────────────┘
```

- The Merge 后 ETH 日发行量降低 **~88%**
- Base Fee 销毁在网络活跃期使 ETH 进入**净通缩**
- 自 2022 年 9 月以来，以太坊已多次处于通缩状态

## 四、账户模型

### EOA（外部账户）

| 属性 | 说明 |
|------|------|
| 控制者 | 私钥持有者 |
| 能否发起交易 | ✅ 能 |
| 有代码 | ❌ 无 |
| 余额存储 | 在全局状态中 |
| 典型场景 | MetaMask 钱包 |

### 合约账户

| 属性 | 说明 |
|------|------|
| 控制者 | 合约代码逻辑 |
| 能否发起交易 | ❌ 不能（需要 EOA 触发） |
| 有代码 | ✅ 有（EVM 字节码） |
| 余额存储 | 在全局状态中 |
| 典型场景 | 代币合约、DEX |

### 两者的关系

```
EOA（人） → 发起交易 → 合约账户（程序）→ 自动执行
                │
         需要 EOA 触发      合约自己不能"主动"做任何事
```

### EIP-7702：边界正在模糊

Pectra 升级（2025 年）引入 EIP-7702，允许 EOA **临时或永久升级为合约账户**，同时保留私钥控制权。这是迈向「账户抽象」的关键一步。
