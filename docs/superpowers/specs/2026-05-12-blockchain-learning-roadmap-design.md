# 区块链工程师学习路线设计方案

## 概述

为有编程基础（TypeScript）的零基础学员设计一条 24 周全职学习路线，目标是在 6 个月内达到区块链开发岗位的求职水平。路线以 TypeScript 为切入点，沿以太坊生态体系展开，涵盖区块链原理、Solidity 合约开发、全栈 DApp 开发、协议精读和求职准备四个阶段。

## 学员画像

| 维度 | 现状 |
|------|------|
| 区块链基础 | 零基础，未接触过区块、共识、智能合约等概念 |
| 编程经验 | 有编程经验，熟悉 TypeScript 等语言 |
| 主攻方向 | TypeScript → DApp / Web3 前端 + 合约开发 |
| 学习目标 | 转行求职，6 个月内具备岗位胜任力 |
| 投入时间 | 全职，每天 6-8 小时 |
| 学习风格 | 项目驱动，理论与实践交替，厌恶长期无产出 |

## 总体路线图

```
第 1-3 周  │ 阶段一：区块链核心概念   │ 交付物：TS 迷你链 + 笔记
第 4-8 周  │ 阶段二：Solidity + Mini   │ 交付物：5 个合约项目 (Hardhat)
第 9-16 周 │ 阶段三：全栈 DApp         │ 交付物：2 个完整 DApp
第 17-24 周│ 阶段四：求职冲刺           │ 交付物：简历 + 作品集 + 面试
```

每个阶段末尾都有可量化的交付物，确保学习进度可追踪、作品集可积累。

## 阶段一：区块链核心概念（第 1-3 周）

### 目标
三周后能用自己的语言完整解释区块链的工作原理，并用 TypeScript 实现核心组件。

### 第一周：区块、链与共识机制

| 天数 | 主题 | 内容 |
|------|------|------|
| 第 1-2 天 | 区块结构 | 区块的构成：交易列表、时间戳、Nonce、父区块哈希、默克尔根 |
| 第 3-4 天 | 哈希函数 | SHA256 原理，"不可逆"和"雪崩效应"，用代码验证 |
| 第 5-6 天 | 链式结构与 PoW | 最长链原则、分叉解决、51% 攻击场景分析 |
| 第 7 天 | 实践 + 总结 | 用 TypeScript 实现一个约 100 行的迷你区块链 |

### 第二周：共识机制与密码学基础

| 天数 | 主题 | 内容 |
|------|------|------|
| 第 1-2 天 | 共识机制对比 | PoW vs PoS vs DPoS vs PBFT，各自的优缺点和适用场景 |
| 第 3-4 天 | 非对称加密 | ECDSA：公私钥对生成、数字签名与验签、地址推导 |
| 第 5-6 天 | 默克尔树 | 默克尔树结构、轻节点如何验证交易、SPV 证明 |
| 第 7 天 | 实践 + 总结 | 用 TypeScript 实现默克尔树验证逻辑 |

### 第三周：以太坊与智能合约入门

| 天数 | 主题 | 内容 |
|------|------|------|
| 第 1-2 天 | 以太坊核心概念 | EVM、智能合约、Gas 机制、账户模型（EOA vs 合约账户） |
| 第 3-4 天 | 交易生命周期 | 创建 → ABI 编码 → ECDSA 签名 → RLP 编码 → 广播 → 打包 → 确认 |
| 第 5-6 天 | 第一个合约 | 使用 Remix IDE 编写和部署 Solidity 合约（存储/读取数据） |
| 第 7 天 | 复盘 | 阶段总结 + 发布第一篇技术笔记（GitHub 或博客） |

### 阶段一交付物
- [ ] TypeScript 迷你区块链实现（GitHub 仓库）
- [ ] TypeScript 默克尔树验证实现
- [ ] Sepolia 测试网部署的第一个合约交易记录
- [ ] 至少 3 篇区块链技术笔记

## 阶段二：Solidity 深入与 Mini 项目（第 4-8 周）

### 目标
彻底掌握 Solidity，能独立设计、编写、测试和部署安全的智能合约。

### 核心知识点（贯穿五周）

- Solidity 类型系统（值类型、引用类型、映射、结构体、枚举）
- 继承、抽象合约、接口、库
- Storage / Memory / Calldata 数据位置与 Gas 影响
- Modifier、Event、Error（require / revert / assert）
- 收发 Ether：payable、fallback、receive
- OpenZeppelin 标准合约库（Ownable、ERC20、ERC721、ReentrancyGuard）
- Hardhat 环境搭建：编译、测试、部署脚本
- Gas 优化基础：变量打包、short-circuit、unchecked 块

### Mini 项目安排

| 周次 | 项目 | 核心知识点 |
|------|------|-----------|
| 第 4 周 | ERC20 代币合约 | 标准接口、铸造/销毁、Transfer 事件、授权机制 |
| 第 5 周 | NFT 铸造合约（ERC721） | 元数据 URI、铸造函数、SafeMint、OpenZeppelin 集成 |
| 第 6 周 | 拍卖合约 | 竞拍出价、最高标管理、资金托管、"拍卖撤回"模式 |
| 第 7 周 | 简易 AMM DEX | 恒定乘积公式（x*y=k）、流动性添加/移除、swap |
| 第 8 周 | DAO 投票合约 | 委托投票、提案创建与执行、法定人数、AccessControl |

### 安全红线（每个合约必须自查）

1. 是否存在重入攻击风险？（是否遵循"检查-生效-交互"模式）
2. 整数运算检查了吗？（Solidity 0.8+ 默认检查溢出，但业务逻辑是否正确？）
3. 权限控制是否足够严格？（多签、时间锁、角色分离）
4. 用户资金是否有被锁死的可能？
5. 是否依赖了可被操纵的随机数或时间戳？

### 阶段二交付物
- [ ] 一个完整的 Hardhat 工程仓库，包含 5 个经过测试的合约
- [ ] 每个合约已部署到 Sepolia 测试链并验证
- [ ] 合约测试覆盖率 80%+
- [ ] 每个合约配有 README 说明文档

## 阶段三：全栈 DApp 开发（第 9-16 周）

### 目标
掌握 DApp 全栈开发能力，能独立完成连接前端 → 智能合约 → 链下索引的完整应用。

### 第九周：前端工具链

- **Wagmi**：React Hooks 框架，替代过时的 ethers.js
- **viem**：轻量级 TypeScript 库，类型安全地读写链上数据
- **RainbowKit / Web3Modal**：接入钱包连接 UI，支持 MetaMask、WalletConnect、Coinbase Wallet
- **项目脚手架**：基于 Wagmi CLI 或 Next.js + RainbowKit 模板初始化项目

### 第十周：DApp 核心交互

- **交易流程管理**：发送交易 → 等待确认 → 处理回执 → 错误处理。含进度 UI（待签名、已广播、已打包、已确认）
- **事件监听**：useWatchContractEvent 实时捕获合约事件，驱动前端状态更新
- **The Graph 索引**：创建 Subgraph，用 GraphQL 替代直接 RPC 查询

### 第十一至十二周：实战项目一 —— 流动性质押协议

**项目描述**：用户存入 ETH，合约铸造等量的流动性质押凭证代币（类似 Lido），协议将资金汇集并展示收益统计。

**功能范围**：
- 前端看板：总锁仓量（TVL）、年化收益率（APR）、用户持仓
- 核心交互：存入 ETH → 获得 stETH 凭证、赎回 stETH → 取回 ETH
- 收益模拟：基于预设策略利率展示累计收益

**技术栈**：React / Next.js + Wagmi + viem + RainbowKit + Hardhat + The Graph

**部署目标**：合约部署到 Sepolia 测试网，前端部署 Vercel，Subgraph 部署到 Subgraph Studio

### 第十三至十六周：实战项目二（选择其一）

| 项目 | 核心挑战 |
|------|---------|
| NFT 市场 | 上架列表、挂单/报价匹配、版税分配、离线签名 |
| 去中心化期权协议 | 期权定价、行权逻辑、抵押品管理 |
| 链上任务平台 | 任务发布与抵押、工作者申请与验收、自动打款 |

### 协议精读作业

- 精读 **Uniswap V3** 核心合约源码，理解架构和主流程
- 阅读一份顶级安全公司的审计报告（如 Trail of Bits 或 ConsenSys Diligence 的公开报告）
- 归纳编写 **"合约安全检查清单"**，形成自己的安全规范

### 阶段三交付物
- [ ] 两个完整全栈 DApp，可在线访问
- [ ] GitHub 仓库包含前端 + 合约 + 测试的完整代码
- [ ] 已部署的 Subgraph 索引服务
- [ ] 协议精读笔记与个人安全检查清单

## 阶段四：求职冲刺（第 17-24 周）

### 目标
系统准备面试，打磨简历和作品集，拿到至少一个区块链开发岗位的面试和 Offer。

### 第十七至十八周：面试知识点梳理

#### 区块链基础面试题库

1. 解释区块结构和链式存储原理
2. 比特币和以太坊的核心区别
3. EVM 是如何执行智能合约的？
4. Gas 机制的设计目的是什么？Gas 消耗如何计算？
5. 分片、Layer 2、Rollup（Optimistic / ZK）分别解决了什么问题？
6. 什么是 MEV？对普通用户和协议分别有什么影响？
7. ECDSA 签名验证流程是怎样的？
8. 什么是最终性（Finality）？不同共识的最终性有何不同？

#### Solidity 与安全面试题库

1. Storage 布局规则：状态变量如何映射到插槽？
2. 常见攻击手段及防护：重入攻击、闪电贷攻击、预言机操纵、三明治攻击、授权钓鱼
3. 代理合约与可升级模式对比：UUPS vs Transparent Proxy vs Beacon Proxy
4. ERC20 / ERC721 / ERC1155 标准的核心接口和使用场景
5. EIP-1559 费用市场改革的内容和影响
6. 合约间交互方式：call / delegatecall / staticcall 的区别和风险
7. 如何处理前端交易失败的回滚？

### 第十九至二十周：简历与作品集打磨

**简历策略**：
- 突出已有的编程经验，包装为"具备 TypeScript 工程能力的区块链开发者"
- 用 STAR 法则描述项目经历（情境-任务-行动-结果）
- 量化成果：如"实现了一个 AMM 交易对池，覆盖 90%+ 测试率"
- 列出技术栈标签：Solidity、Hardhat、Wagmi、React、TypeScript、The Graph

**作品集页面**：一个简洁的 Landing Page，每个项目呈现：演示链接、GitHub 链接、一句话说明、技术栈标签

**技术博客**：选 2-3 个深入研究的主题写成文章，发表于 Mirror.xyz 或自建博客

### 第二十一至二十四周：投递与面试实战

**目标渠道**：
- 电鸭社区、远程 OK 等中文远程工作平台
- Cryptocurrency Jobs、CryptoJobsList
- Gitcoin 悬赏任务（积累链上履历）
- 项目方官网招聘页 / Twitter / Farcaster 生态招聘帖
- 目标投递量：20-30 份

**模拟面试**：
- 每周至少一次模拟面试
- 内容涵盖：基础概念问答 + 合约设计白板 + 系统设计讨论
- 每次模拟后给出改进建议

**链上实战面试准备**：
- 部分岗位有现场编码环节（"根据需求写一个合约"）
- 重点训练：5 分钟内理清需求 → 写出安全合约 → 讲清设计思路
