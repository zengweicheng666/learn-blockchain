# 区块链学习计划 · 阶段四：求职冲刺（第 17-24 周）

> **说明：** 这个阶段的核心是「把积累转化为面试表现」。前两周系统梳理面试题，中间两周打磨简历和作品集，最后四周实战投递和模拟面试。

**目标：** 投递 20+ 岗位，通过面试拿到 Offer

**前置条件：** 已完成前三个阶段，有 5+ 合约项目 + 2 个完整 DApp 的作品集

---

## 第十七至十八周：面试知识点系统梳理

### 第十七周：区块链基础面试题

- [ ] **第 1 天：区块 + 链 + 共识**
  - 准备答案：解释区块结构和链式存储原理
  - 准备答案：比特币和以太坊的核心区别（3 点以上）
  - 准备答案：PoW vs PoS 对比
  - 演练：对着镜子或者录音讲一遍，时间控制在 2 分钟以内
  - **产出：** 面试题库答案 Q1-Q3

- [ ] **第 2 天：EVM + Gas**
  - 准备答案：EVM 是如何执行智能合约的？（从 opcode 到执行流程）
  - 准备答案：Gas 机制的设计目的？Gas 消耗如何计算？
  - 准备答案：EIP-1559 前后费用市场的区别
  - **产出：** 面试题库答案 Q4-Q6

- [ ] **第 3 天：Layer 2 + 扩容**
  - 准备答案：分片、Layer 2、Rollup（Optimistic / ZK）分别解决了什么问题？
  - 准备答案：Optimistic Rollup vs ZK Rollup 对比
  - 实际跑一下：在 Arbitrum Sepolia 或 Optimism Sepolia 上发一笔交易
  - **产出：** 面试题库答案 Q7-Q9 + L2 交易记录

- [ ] **第 4 天：MEV + 安全概念**
  - 准备答案：什么是 MEV？对普通用户有什么影响？
  - 准备答案：什么是最终性？不同共识的最终性有何不同？
  - 准备答案：ECDSA 签名验证流程
  - **产出：** 面试题库答案 Q10-Q12

- [ ] **第 5 天：综合演练**
  - 随机抽 5 道题，限时回答（每题 3 分钟）
  - 录音回听，找可以改进的地方
  - **产出：** 自我评估记录

- [ ] **第 6-7 天：背题 + 复盘**
  - 反复朗读答案直到自然流畅
  - 找到 3 个"个人亮点"故事：选你项目中最有成就感的部分，准备 1 分钟版本

### 第十八周：Solidity / 安全面试题

- [ ] **第 1 天：Storage + Gas**
  - 准备答案：Storage 布局规则——状态变量如何映射到插槽？
  - 准备答案：Storage / Memory / Calldata 的区别
  - 准备答案：Gas 优化技巧（变量打包、unchecked、short-circuit）
  - 练：在纸上写出一个合约的 Storage 布局
  - **产出：** 题库答案 Q13-Q15

- [ ] **第 2 天：常见攻击与防护**
  - 准备答案：重入攻击的原理和防护（Check-Effects-Interact 模式 + ReentrancyGuard）
  - 准备答案：闪电贷攻击的原理和案例
  - 准备答案：预言机操纵和三明治攻击
  - 练：在纸上写出一个带重入防护的取款函数
  - **产出：** 题库答案 Q16-Q19

- [ ] **第 3 天：代理合约 + 可升级模式**
  - 准备答案：为什么需要可升级合约？
  - 准备答案：UUPS vs Transparent Proxy vs Beacon Proxy 的区别
  - 准备答案：delegatecall 的工作原理和风险
  - 练：在纸上画出 UUPS 代理的架构图
  - **产出：** 题库答案 Q20-Q22

- [ ] **第 4 天：ERC 标准 + 系统设计**
  - 准备答案：ERC20 / ERC721 / ERC1155 标准和接口对比
  - 准备答案：合约间交互方式（call / delegatecall / staticcall）
  - 练：画出一个 DEX 系统的整体架构图
  - **产出：** 题库答案 Q23-Q25

- [ ] **第 5 天：模拟面试（我出题你答）**
  - 我会在终端里随机出 8-10 道面试题
  - 你回答，我给你反馈（逻辑清晰度、完整性、表达）
  - 记录薄弱环节
  - **产出：** 模拟面试记录 + 薄弱点清单

- [ ] **第 6-7 天：回炉薄弱点**
  - 针对薄弱环节重新收集资料和整理答案
  - 整体过一遍所有面试题，确保每道都能流利回答
  - **产出：** 更新后的完整面试题库

## 第十九至二十周：简历 + 作品集打磨

### 第十九周：简历制作

- [ ] **第 1 天：简历框架**
  - 下载或打开简历模板（推荐用简约排版，一页为限）
  - 写下个人信息 + 自我介绍（2-3 句，突出"有编程基础 + 区块链专项学习"）
  - **产出：** 简历草稿 V1

- [ ] **第 2 天：项目经验 —— 选择 & 描述**
  - 挑选 2-3 个最强项目放进简历
  - 用 STAR 法则写每个项目的描述：
    - Situation：背景是什么
    - Task：要解决什么问题
    - Action：你具体做了什么（技术选型、架构决策、实现的难点）
    - Result：结果和量化数据（测试覆盖率、部署地址、用户量等）
  - **产出：** 简历项目描述部分

- [ ] **第 3 天：技能列表 + 排版**
  - 技能标签：Solidity、Hardhat、Wagmi、viem、React、TypeScript、The Graph、Ethereum、EVM、Layer 2
  - 排版优化：一页之内，清晰可读
  - **产出：** 简历 V1 完整版

- [ ] **第 4 天：简历迭代**
  - 找 1-2 个朋友同事帮忙看看简历
  - 收集反馈并修改
  - **产出：** 简历 V2

- [ ] **第 5-7 天：作品集页面**
  - 用 Next.js 做一个简洁的作品集 Landing Page（或用已有的 DApp 前端改）
  - 每个项目展示：截图、一句话介绍、技术栈标签、演示链接、GitHub 链接
  - 部署到 Vercel，绑定自定义域名（如果有）
  - **产出：** 在线作品集页面

  - 把最有代表性的项目置顶到 GitHub Profile

- [ ] **第 5-7 天：投递准备**

  - 写求职信草稿模板（可随岗位调整）
  - **产出：** 投递跟踪表 + 求职信模板

### 第二十二周：面试实战 + 复盘

- [ ] **第 1-2 天：第一轮面试**
  - 如果有面试邀请，按时间参加
  - 每场面试后记录：被问了什么、哪里答得好、哪里没答好
  - **产出：** 面试记录复盘

- [ ] **第 3-4 天：继续投递 + Gitcoin**
  - 完成 Gitcoin Bounty 任务（如果上周接的还没做完）
  - 继续投递 5-10 个岗位（补充新鲜 JD）
  - **产出：** 累计投递量 20-25 个

- [ ] **第 5 天：模拟面试**
  - 再次模拟面试，这次着重练"即兴设计"类问题（"如果让你设计一个 XX 协议，你怎么做？"）
  - **产出：** 模拟面试记录

- [ ] **第 6-7 天：复盘 + 策略调整**
  - 分析前两周的投递数据：多少已读、多少回复、多少面试
  - 如果回复率低于 10%，优化简历和目标岗位的匹配度
  - 如果面试通过率低，针对性补弱
  - **产出：** 调整后的求职策略

### 第二十三至二十四周：冲刺 + 收尾

- [ ] **第 1-2 天：继续投递**

  - **产出：** 限时编程练习记录

  - **产出：** 薪资谈判笔记

- [ ] **最后三天：整体复盘 + 下一步计划**
  - 回顾 24 周的学习历程
  - 整理最终的作品集和 GitHub
  - 无论是否已拿到 Offer，制定下一步成长计划
  - **产出：** 个人成长总结 + 下一阶段计划

## 执行建议

1. **每天开工前**，看一眼当天的任务清单，预估时间
2. **每周末**，用复盘检查清单确认进度
3. **不要跳过测试**——测试是你理解合约的印证，也是面试时的加分项
4. **每完成一个项目**，发一条推或记一篇笔记，积累影响力
5. **遇到卡壳**，随时找我，我们可以一起 debug 或讨论设计问题
