# 区块链学习计划 · 阶段三：全栈 DApp 开发（第 9-16 周）

> **说明：** 从纯合约开发进入全栈 DApp 开发，学会前端 + 合约 + 链下索引的完整技术栈。第 9-10 周为工具学习，第 11-16 周为实战项目。

**目标：** 独立完成两个完整的全栈 DApp，可在线访问

**前置条件：** 已完成阶段一和阶段二，熟悉 Solidity 和 Hardhat

---

## 第九周：前端工具链（Wagmi + viem + RainbowKit）

### 第 1-2 天：Wagmi 框架入门

- [ ] **第 1 天：环境搭建**
  - 学：用 `npm create wagmi@latest` 创建项目，选择 React + TypeScript 模板
  - 理解 Wagmi 核心概念：WagmiProvider、Hooks 模式、自动重新连接
  - 搭 Hello World：页面显示 "连接钱包" 按钮 + 连接后显示地址
  - **产出：** `projects/dapp/` 脚手架，钱包连接功能可用

- [ ] **第 2 天：RainbowKit + 多钱包支持**
  - 集成 RainbowKit：支持 MetaMask、WalletConnect、Coinbase Wallet
  - 自定义主题色和按钮文案
  - 测试：切换不同钱包、断开连接、切换链
  - **产出：** 多钱包连接 UI 已完成

### 第 3-4 天：viem 与合约交互

- [ ] **第 3 天：读合约数据**
  - 学：viem 基础 —— createPublicClient、readContract、getBalance
  - 练：连接 Sepolia 测试网，读一个 ERC20 代币的 name、symbol、balanceOf
  - 用 Wagmi 的 `useReadContract` hook 封装读取逻辑
  - **产出：** 前端页面展示 ERC20 代币信息

- [ ] **第 4 天：写合约（交易）**
  - 学：Wagmi 的 `useWriteContract` 和 `useWaitForTransactionReceipt`
  - 练：写一个 transfer 功能界面：输入接收地址和金额 → 发送交易 → 等待确认 → 显示结果
  - 处理交易状态：待签名、已提交、已确认、失败
  - **产出：** 前端 token 转账功能可用

### 第 5-7 天：The Graph 索引 + 综合练习

- [ ] **第 5 天：Subgraph 基础**
  - 学：The Graph 是什么（去中心化索引协议）
  - 安装 Graph CLI：`npm install -g @graphprotocol/graph-cli`
  - 初始化一个 Subgraph，连接你的 ERC20 合约
  - 定义 schema（Token、Transfer 实体）+ 映射转账事件
  - **产出：** Subgraph 项目脚手架，成功索引 Transfer 事件

- [ ] **第 6 天：Subgraph 部署**
  - 在 Subgraph Studio 创建 API Key
  - 部署 Subgraph 到 Subgraph Studio 的测试网
  - 写一个 GraphQL 查询：查询某地址最近的 10 笔转账
  - 在前端中用 `urql` 或 `graphql-request` 调用 Subgraph
  - **产出：** 已部署的 Subgraph + 前端 GraphQL 查询

- [ ] **第 7 天：本周复盘**
  - 整理本周学习笔记
  - 确保所有代码推送 GitHub
  - **检查清单：**
    - [ ] Wagmi + RainbowKit 钱包连接已跑通
    - [ ] 前端读/写合约功能正常
    - [ ] Subgraph 已部署并返回数据
    - [ ] 理解 DApp 全栈架构（前端 ↔ Subgraph ↔ RPC ↔ 合约）

## 第十周：DApp 核心交互进阶

### 第 1-2 天：交易 UX 优化

- [ ] **第 1 天：交易状态组件**
  - 实现交易进度 UI 组件：待签名（钱包弹窗）、已广播（pending）、已确认（success）、失败（error with tx hash）
  - 实现交易历史列表组件（从 Subgraph 查询）
  - 实现友好的错误提示：将原始的 revert reason 转成中文可读消息
  - **产出：** 可复用的交易状态组件

- [ ] **第 2 天：事件监听 + 实时更新**
  - 用 Wagmi 的 `useWatchContractEvent` 实时监听合约事件
  - 监听到新事件后自动更新 UI（如转账后余额自动刷新）
  - 对比：轮询 vs WebSocket vs 事件监听
  - **产出：** 实时更新的交易列表和余额

### 第 3-7 天：准备项目一的技术组件

- [ ] **第 3 天：合约开发 —— 流动性质押合约**
  - 写 `LiquidStaking.sol`：接收 ETH，铸造 stETH 凭证代币
  - 功能：`stake()` → ETH in, stETH out；`unstake(uint amount)` → stETH in, ETH out
  - 合约收益模拟：预设一个年化收益率，累积展示
  - 写 Hardhat 测试
  - **产出：** 质押合约 + 测试

- [ ] **第 4 天：合约安全 + 部署**
  - 安全检查：重入（stake/unstake 顺序）、价格操纵、ETH 精度
  - 部署到 Sepolia 测试网 + etherscan 验证
  - 写部署脚本 + 验证脚本
  - **产出：** 已部署的质押合约

- [ ] **第 5 天：Subgraph 对接质押合约**
  - 为质押合约创建 Subgraph：索引 Stake、Unstake、RewardClaimed 事件
  - 部署 Subgraph 到 Subgraph Studio
  - **产出：** 质押合约的 Subgraph 已部署

- [ ] **第 6-7 天：本周复盘**
  - 整理笔记，确保代码已推送
  - **检查清单：**
    - [ ] 交易进度组件可复用
    - [ ] 实时事件监听已实现
    - [ ] 质押合约开发完成、测试通过、已部署
